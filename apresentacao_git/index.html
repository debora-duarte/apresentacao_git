O que é o Git?

O Git é um sistema de controle de versão distribuído.

Mas o que é isso?

É um sistema que usamos com dois grandes propósitos:

  • Facilitar o trabalho em equipe
  • Fazer o versionamento de um trabalho

Como funciona geralmente?

Repositório remoto

Implemento alterações

Faço merge*

Feature pronta

Git clone: fazer uma cópia de um repositório remoto já existente.

Git init: inicializa um repositório vazio localmente.

Git branch: uma ramificação do trabalho do grupo para que, por exemplo, o desenvolvimento de uma nova feature possa seguir em paralelo com o branch de produção. Assim, não existem o risco de ser necessário corrigir um bug em produção e o código estar todo misturado.

Git commit: envio das alterações para seu repositório local.

Merge: resolução manual de conflitos de trabalhos feitos por diferentes desenvolvedores em um mesmo arquivo.

Git merge branch: junção dos trabalhos de dois branchs diferentes. Nesse processo, podem existir conflitos em alguns arquivos que precisam ser resolvidos manualmente, como descrito acima.

Git na prática

Boas práticas

  • Commits : de alterações que representem uma tarefa
  • Branchs : novas features no projeto
  • Pull requests : revisão do trabalho antes do merge dos branchs
